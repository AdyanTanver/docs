## Overview

The tenant screening flow API enables property managers to conduct thorough background checks on potential tenants. This guide explains how to implement each verification node in the screening process.

## Base URL

```bash
https://sandbox-api.getcove.co/v1/flows/tenant-screening
```

## Node Implementation

### 1. ID Verification Node

#### Endpoint

```bash
POST /nodes/idv
```

#### Request Body

```json
{
  "node_id": "id_verification_v1",
  "provider": "Persona",
  "options": {
    "require_selfie": true,
    "enable_liveness_detection": true,
    "require_id_back_side": true,
    "check_id_expiration": true
  }
}
```

#### Response

```json
{
  "status": "completed",
  "result": {
    "verification_id": "ver_123456",
    "id_valid": true,
    "identity": {
      "first_name": "John",
      "last_name": "Doe",
      "date_of_birth": "1990-01-01"
    }
  }
}
```

[View IDV Node Details](/steps/idv)

## Complete Flow Implementation

### Create Flow

#### Endpoint

```bash
POST /flows/tenant-screening
```

#### Request Body

```json
{
  "flow_id": "tenant_screening_v1",
  "nodes": [
    {
      "node": "ID_VERIFICATION",
      "required": true,
      "provider": "Persona"
    },
    {
      "node": "PERSONAL_INFO",
      "required": true
    },
    {
      "node": "EMPLOYMENT",
      "required": true,
      "provider": "Argyle",
      "fallback_provider": "Atomic"
    },
    {
      "node": "BANK_CONNECT",
      "required": true,
      "providers": ["Plaid", "Flinks", "MX", "Fincity", "Inverite"]
    },
    {
      "node": "RESIDENCE_HISTORY",
      "required": true
    },
    {
      "node": "ASSETS",
      "required": true,
      "provider": "Plaid"
    },
    {
      "node": "CREDIT_REPORT",
      "required": true,
      "provider": "Equifax",
      "fallback_provider": "TransUnion"
    },
    {
      "node": "BACKGROUND_CHECK",
      "required": true,
      "provider": "Persona"
    }
  ]
}
```

### Get Flow Status

#### Endpoint

```bash
GET /flows/tenant-screening/{flow_id}
```

#### Response

```json
{
  "flow_id": "tenant_screening_v1",
  "status": "in_progress",
  "completed_nodes": ["ID_VERIFICATION", "PERSONAL_INFO"],
  "current_node": "EMPLOYMENT",

    "BANK_CONNECT",
    "RESIDENCE_HISTORY",
    "ASSETS",
    "CREDIT_REPORT",
    "BACKGROUND_CHECK"
  ]
}
```

## Error Handling

All API endpoints return standard HTTP status codes:

- `200`: Success
- `400`: Bad Request
- `401`: Unauthorized
- `403`: Forbidden
- `404`: Not Found
- `429`: Too Many Requests
- `500`: Internal Server Error

Error responses include detailed messages:

```json
{
  "error": {
    "code": "invalid_request",
    "message": "Invalid provider configuration",
    "details": {
      "field": "provider",
      "reason": "Unsupported provider for this node type"
    }
  }
}
```
