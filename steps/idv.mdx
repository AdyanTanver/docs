---
title: "ID Verification"
description: "The ID verification step allows for automated verification of government-issued identification documents and performs background checks."
---

## Configuration

```json
{
    "id": "6797df4ca4c4910bc0f1d48f",
    "name": "ID_VERIFICATION",
    "options": {
        "require_selfie": true,
        "enable_liveness_detection": true,
        "require_id_back_side": false,
        "run_background_check": {
            "watchlist_check": true,
            "sanctions_check": true
        }
    },
    "control_flow": {
        "condition": {
            "condition_operator": "==",
            "condition_field": "id_valid",
            "condition_value": true
        },
        "if_goto": "6797dfe7a4c4910bc0f1d492",
        "else_goto": "6797fd5e683fb54e8abe7d852"
    },
    "provider": ["Persona"]
}
```

## Config Fields

### Required Fields
- `id`: Unique identifier for the verification step
- `name`: Step type identifier (ID_VERIFICATION)

### Options
- `require_selfie`: Whether to require a selfie photo for identity verification
- `enable_liveness_detection`: Whether to perform liveness checks to prevent fraud
- `require_id_back_side`: Whether to require both sides of the ID document
- `run_background_check`: Configuration for background check options
  - `watchlist_check`: Whether to check against watchlists
  - `sanctions_check`: Whether to check against sanctions lists

### Control Flow
Determines the next step based on verification results:
- If `id_valid` is `true`, proceeds to step `6797dfe7a4c4910bc0f1d492`
- Otherwise, goes to step `6797fd5e683fb54e8abe7d852`

### Providers
Uses Persona for both ID verification and background checks.

```json
        "status": "COMPLETED",

            "number": "DL123456789"
        },
        "identity": {
            "name": {
                "first": "John",
                "last": "Doe"
            },
            "date_of_birth": "1990-01-15",
            "address": {
                "street": "123 Main St",
                "city": "Toronto",
                "provinceState": "ON",
                "postalCode": "M5V 2T6"
            }
        },
        "verification": {
            "selfie_match": true,
            "liveness_check": true
        },
        "background_check": {
            "watchlist_hit": false,
            "sanctions_hit": false
        }
    }

```

## Output Fields

### Status Information
- `status`: Current status of the verification (e.g., "COMPLETED")
- `id_valid`: Whether the ID was successfully verified

### Document Information
- `document`: Details about the verified ID document
  - `type`: Type of ID document provided
  - `expiration_date`: When the document expires
  - `number`: Document identification number

### Identity Information
- `identity`: Extracted identity details
  - `name`: First and last name
  - `date_of_birth`: Date of birth from ID
  - `address`: Current address information

### Verification Results
- `verification`: Results of verification checks
  - `selfie_match`: Whether the selfie matched the ID photo
  - `liveness_check`: Whether the liveness check passed

### Background Check Results
- `background_check`: Results of background screening
  - `watchlist_hit`: Whether any watchlist matches were found
  - `sanctions_hit`: Whether any sanctions matches were found
